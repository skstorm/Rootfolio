# Rootfolio 프로젝트 규칙 (번역본)

이 문서는 Rootfolio 프로젝트의 영구 지시 사항인 `rules.md`의 한국어 번역본입니다.

## 📋 언어 및 출력 규칙

- **사용자 소통**: 모든 설명, 진행 상황 업데이트, 사용자 알림 등은 **한국어**로 작성합니다.
- **소스 코드**: 파일 내의 코드 주석 및 설명은 **한국어**로 작성합니다.
- **설정 파일**: AI 에이전트용 파일(`.agent/instructions/` 내 파일 등)은 AI의 정확한 이해를 위해 **영어**로 작성합니다.
- **번역 요구사항**: 모든 영어 설정 파일은 반드시 `파일명(korea).md` 형식의 한국어 번역본 세트를 가져야 합니다. (예: `rules.md` -> `rules(korea).md`)
- **AI 작동 로직**: AI 에이전트는 프로젝트 지시 사항이나 규칙을 읽을 때 파일명에 `(korea)`가 포함된 모든 파일을 **무시**해야 합니다. 이 파일들은 오직 사용자를 위한 것입니다.

## 📂 프로젝트 아키텍처

- **구조 엄수**: 다음의 계층 구조를 유지합니다:
    - `platform/`: 메인 허브 사이트 소스 코드.
    - `content/`: 학습 노트(`studies/`) 및 일일 로그(`logs/`).
    - `modules/apps/`: 통합된 외부 앱 (예: Flutter 프로젝트).
- **형식**: 지식 기록 및 로그는 **마크다운(Markdown)** 형식을 기본으로 합니다.

## 🛠️ 개발 스타일
- 현대적이고 깔끔하며 모듈화된 코딩 패턴을 사용합니다.
- 모든 자동화 작업은 프로젝트의 디자인 가이드라인(다크 모드, 프리미엄 UI)을 따릅니다.
