# 작업일지 작성 워크플로우

작업 세션이 끝난 후 반드시 실행하여 기록을 남긴다.

---

## Step 1. 파일 생성

- 파일 경로: `e:\Study\Rootfolio\docs\WorkLog\WorkLog_[오늘날짜].md`
- 날짜 형식: `YYYY-MM-DD` (예: `WorkLog_2026-03-13.md`)
- 이미 파일이 있으면 해당 파일을 이어서 수정한다.

---

## Step 2. 파일 구조 (아래 템플릿을 그대로 사용)

```markdown
# 작업 일지 (YYYY-MM-DD)

## 🎯 오늘의 목표
- 목표 1
- 목표 2

---

## ✅ 완료된 작업

### [작업 제목]
- 작업 내용 요약
- 생성/수정된 주요 파일 목록

---

## 🔥 트러블슈팅

### [문제 제목]
- **원인**: 문제가 발생한 이유
- **해결**: 어떻게 해결했는지
- **교훈**: (선택) 이 문제에서 얻은 인사이트

---

## 🛡️ 내일 할 일 (Next Steps)
- [ ] 다음 작업 1
- [ ] 다음 작업 2
```

---

## Step 3. 작성 규칙

1. **완료된 작업** 섹션
   - 생성/수정한 주요 파일명을 백틱으로 기재
   - 어떤 기능을 만들었는지 한 줄씩 기술
   - 오늘 한 커밋 목록 포함 (있는 경우)

2. **트러블슈팅** 섹션 ← 반드시 포함
   - 에러 메시지, 빌드 오류, 예측 못 한 문제 등 모두 기록
   - 문제가 없었던 날도 `"특이사항 없음"` 으로 명시
   - 원인 / 해결 / 교훈 구조를 반드시 지킨다

3. **내일 할 일** 섹션
   - 체크박스(`- [ ]`) 형식으로 작성
   - 오늘 못 한 것, 다음 Phase, 알아봐야 할 것 포함

---

## Step 4. 저장 및 확인

작성 후 파일을 저장하고 사용자에게 파일 경로를 알려준다.
