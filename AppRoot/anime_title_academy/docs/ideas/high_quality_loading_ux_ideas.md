# 고품질 자막 생성 대기 UX 아이디어

## 배경
- `고품질` 모드는 `gemini-2.5-pro`를 사용하므로 응답 시간이 길어질 수 있다.
- 병목은 주로 LLM 응답 시간이며, 클라이언트에서 직접 줄일 수 있는 범위에는 한계가 있다.
- 따라서 대기 시간을 숨기기보다, 사용자가 지루하지 않게 느끼도록 UX를 설계하는 것이 중요하다.

## 목표
- 긴 대기 시간을 "버벅임"이 아니라 "의도된 고품질 생성 과정"처럼 느끼게 한다.
- 사용자가 현재 AI가 무엇을 하고 있는지 이해할 수 있게 한다.
- 결과가 나왔을 때 기다린 보상이 있다고 느끼게 한다.

## 아이디어

### 1. 애니 연출형 로딩
- 단순 스피너 대신 단계형 메시지를 순환 표시한다.
- 예시 문구:
  - `장면의 분위기를 해석하고 있습니다`
  - `감정선에 맞는 제목을 고르고 있습니다`
  - `더 인상적인 문장으로 다듬고 있습니다`
- 배경 이미지 위에 은은한 페이드, 스캔 라인, 타자기 느낌 텍스트 등 가벼운 연출을 추가한다.

### 2. 분석 태그 선공개
- Vision 단계가 끝난 후에는 추출된 태그를 먼저 보여준다.
- 예:
  - `노을`
  - `학교`
  - `청춘`
  - `옥상`
- 사용자는 AI가 어떤 재료를 바탕으로 제목을 만들고 있는지 볼 수 있다.

### 3. 제목 후보 실루엣 연출
- 실제 결과 전에는 흐릿한 placeholder 문구를 보여준다.
- 예:
  - `......의 거리`
  - `너와 나의 ......`
  - `깨어나는 ......`
- 실제 결과가 아니라 "제목을 조합 중"이라는 연출용 요소로 사용한다.

### 4. 고품질 모드 전용 안내 카피
- 사용자가 고품질 모드의 느린 속도를 자연스럽게 받아들이도록 짧은 설명을 넣는다.
- 예:
  - `고품질 모드는 더 오래 생각합니다`
  - `표현과 감정선을 더 정교하게 다듬는 중입니다`

### 5. 대기 중 미니 인터랙션
- 생성 중에도 가볍게 볼 수 있는 정보를 제공한다.
- 예:
  - 현재 스타일 설명
  - 이번 생성의 핵심 태그
  - 직전 생성 제목 기록
- 생성 흐름을 방해하지 않는 수준으로만 제공한다.

### 6. 단계형 진행 시각화
- 실제 퍼센트 대신 구간 기반 상태를 보여준다.
- 예:
  - `장면 해석 완료`
  - `제목 구조 생성 중`
  - `문장 다듬는 중`
- 실제 내부 로그와 완전히 일치할 필요는 없지만, 현재 단계처럼 느껴지게 설계한다.

### 7. 결과 보상 강화
- 오래 기다린 만큼 결과 노출 연출을 강화한다.
- 예:
  - 고품질 모드 전용 등장 애니메이션
  - 더 고급스러운 particle/confetti 효과
  - `고품질 생성 완료` 배지

## 추천 조합
- `고품질` 모드 전용 로딩 화면
- 단계형 문구 3개 순환
- Vision 태그 선공개
- 결과 노출 연출 강화

## 구현 우선순위
1. 고품질 모드 여부에 따라 로딩 문구 분기
2. 단계형 메시지 순환 애니메이션 추가
3. Vision 태그 선공개 UI 추가
4. 결과 등장 연출 강화

## 메모
- 이 방향은 서버 응답 시간을 줄이지 못해도 체감 속도를 개선할 수 있다.
- 로딩 UX는 고품질 모드에서만 차별화하고, `빠름`과 `밸런스`는 현재보다 약간만 보강하는 편이 자연스럽다.
