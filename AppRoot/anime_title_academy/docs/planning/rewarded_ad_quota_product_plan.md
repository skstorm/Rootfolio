# 보상형 광고 기반 무료 이용권 기획서

## 문서 목적
- `anime_title_academy`에 광고를 붙일 때, 사용성을 해치지 않으면서 수익화를 시도할 수 있는 기본 정책을 정의한다.
- 본 문서는 `무료 이용권 + 보상형 광고 충전` 구조의 제품 기획 기준을 정리한다.

## 배경
- 현재 앱은 `빠름 / 밸런스 / 고품질` 3가지 LLM 모드를 제공한다.
- 사용자 입장에서 가장 자주 반복하는 행동은 `다시하기`, `자막만 다시 생성`이다.
- 전면광고는 이 반복 흐름과 충돌하기 쉬워 메인 수익화 방식으로 부적합하다.
- 따라서 사용자가 직접 선택하는 `보상형 광고`가 핵심 수익화 방식으로 적합하다.

## 목표
- 첫 사용 경험을 최대한 깨지 않는다.
- 모델별 가치 차등을 유지한다.
- 사용자가 광고를 보는 이유를 명확하게 이해할 수 있게 한다.
- 개발모드에서는 광고 없이도 앱을 불편 없이 사용할 수 있게 한다.

## 핵심 정책

### 1. 무료 이용권
- 무료 이용권은 모델별로 매일 제공된다.
- 매일 `00:00`에 초기화된다.

### 2. 모델별 기본 무료 횟수
- `빠름`: 5회
- `밸런스`: 3회
- `고품질`: 1회

### 3. 보상형 광고 충전 정책
- 무료 이용권이 모두 소진되면 보상형 광고를 통해 사용권을 충전할 수 있다.
- 광고 1회당 충전량:
  - `빠름`: 3회
  - `밸런스`: 2회
  - `고품질`: 2회
- 광고로 충전된 잔여량도 매일 `00:00`에 함께 초기화된다.

### 4. 광고 실패 정책
- 광고 로드 실패, 재생 실패, 사용자 중도 취소 시 충전은 지급하지 않는다.
- 사용자에게 짧은 안내 문구만 제공한다.

### 5. 개발모드 정책
- 개발모드에서는 광고를 띄우지 않는다.
- 개발모드에서는 무료 이용권 제한도 적용하지 않는다.
- 즉 개발 중에는 모든 모델을 광고 없이 자유롭게 테스트할 수 있어야 한다.

## 사용자 경험 원칙

### 1. 광고는 강제가 아니라 교환이다
- 사용자는 `광고를 보면 추가 생성권을 받는다`는 구조를 명확히 이해해야 한다.
- 광고는 벌칙처럼 느껴지면 안 된다.

### 2. 첫 결과 경험은 보호한다
- 앱의 핵심 가치는 사진을 넣고 제목을 받아보는 경험이다.
- 따라서 첫 경험 자체를 광고로 막지 않는다.

### 3. 반복 사용 구간에서만 수익화한다
- 무료 횟수 소진 이후에만 광고가 개입한다.
- 결과 생성 흐름 중간에 광고를 끼워 넣지 않는다.

### 4. 모델별 가치 차이를 유지한다
- `고품질`은 가장 희소하고 가치가 큰 자원처럼 느껴져야 한다.
- `빠름`은 가장 접근성이 좋아야 한다.

## 기본 UX 흐름

### 무료 횟수가 남아 있을 때
1. 사용자가 모델을 선택한다.
2. 남은 횟수를 확인한다.
3. 생성 버튼을 누르면 즉시 1회 차감 후 생성한다.

### 무료 횟수와 충전 횟수가 모두 없을 때
1. 사용자가 해당 모델로 생성을 시도한다.
2. 앱은 `광고 보고 충전` UI를 보여준다.
3. 사용자가 광고 시청을 완료하면 해당 모델의 충전 횟수를 지급한다.
4. 충전 성공 직후 현재 요청한 생성 흐름을 자동으로 이어서 실행한다.

## 표시해야 할 정보
- 현재 선택된 모델
- 해당 모델의 남은 무료 횟수
- 해당 모델의 남은 충전 횟수
- 광고 1회 시청 시 충전되는 횟수

## 데이터 보존 원칙
- 사용량과 충전량은 앱 재실행 후에도 유지되어야 한다.
- 날짜가 바뀌면 자동으로 일일 무료 횟수를 초기화해야 한다.
- 저장소는 `shared_preferences`를 사용한다.

## 추후 변경 가능 항목
- 모델별 무료 횟수
- 모델별 광고 충전량
- 초기화 시각
- 광고 표시 문구
- 무료/충전 잔여량 표시 방식

## 설정 분리 원칙
- 위 값들은 나중에 변경하기 쉽게 별도 상수/설정 파일에서 관리한다.
- UI, 비즈니스 로직, 광고 로직이 각각 임의 숫자를 직접 들고 있으면 안 된다.

## 범위 제외
- 구독 모델
- 전면광고 정책
- 배너광고 정책
- 서버 연동형 원격 설정

## 성공 기준
- 광고가 없어도 개발모드에서 앱이 정상 동작한다.
- 릴리스 모드에서는 무료 이용권과 충전 정책이 일관되게 동작한다.
- 사용자가 광고 시청의 대가를 명확히 이해할 수 있다.
