# 사다리 게임 (Ghost Leg) 앱 개발 현황 및 향후 과제

본 문서는 사다리 게임 애플리케이션의 현재 개발 완료 상태와 보완해야 할 향후 과제를 정리한 문서입니다. 본 문서 작성 시점을 기준으로 **사다리 게임 앱 개발의 1차 마일스톤은 완료**되었으며, Rootfolio 웹 플랫폼 내 하위 앱 구동 및 기본적인 상호작용 체계까지 확립된 상태입니다.

---

## 🟢 1. 현재 개발 완료 현황 (Current Status)

사다리 게임은 Flutter Web 기술을 사용하여 완성되었으며, Rootfolio 내의 모듈 기반 배포 시스템(Deploy Engine)과 성공적으로 연동되어 있습니다.

### 1-1. 핵심 게임 엔진 (Core Engine)
- **동적 가변 사다리 맵 생성**: 입력받은 N개의 인원(시작 슬롯)과 임의의 세로 폭에 맞춰 Y축 충돌을 방지하며 가로선을 균등하게 생성하는 로직 구현.
- **길찾기 알고리즘**: 특정 플레이어에서 출발해 하단 결과 슬롯까지 도달하기 위해 꺾이는 모든 이동 좌표 목록(`List<Offset>`) 계산 완료.

### 1-2. 시각적 렌더링 시스템 (Renderer)
- **`CustomPainter` 프레임워크 도입**: Flutter 위젯 트리 성능 한계 극복 및 고성능 프레임 유지를 위한 커스텀 드로잉 적용.
- **라인 드로잉(Line Drawing) 최적화**: 경로를 사각형 블록에서 `PaintingStyle.stroke`를 활용해 일직선 네온 테마로 시원하게 렌더링.
- **실시간 경로 채우기 애니메이션**: 매끄럽게 선명도를 조절하며 이동 경로가 채워지도록 구현.

### 1-3. 통합 및 사용자 인터페이스 (Integration & UX)
- **시점 고정형 UX 적용**: 애니메이션 동작 시 카메라 흔들림이나 극심한 시야 변경을 배제하여 어지럼증 완화.
- **상호작용 명확화**: 상단 시작 노드 영역에 전원(버튼) 아이콘을 적용해 사용자가 터치할 부분을 직관적으로 파악하도록 개선.
- **모바일 대응 체계**: Rootfolio 통합 시 세로 화면(Mobile)에서도 우측 햄버거 메뉴를 통한 반응형 내비게이션 지원.

### 1-4. 빌드 및 배포 자동화 체계 
- **ESM 기반 배포 엔진 (`DeployEngine`) 호환 완료**: Flutter 하위 앱이 메인 페이지의 특정 영역에서 정상 렌더링되게 `--base-href`를 스크립트로 안전하게 주입.
- **플랫폼 교차 개발 (Cross-Platform Dev)**: `puro flutter` 명령을 npm scripts에 주입해, 메인 웹 개발과 Flutter 연동 테스트를 동시에 진행하는 환결 정립.

## 🟡 2. 향후 과제 및 개선 포인트 (Future Work)

최초 설계 기획안(`ladder_game_critique.md`)을 바탕으로 더 '게임다운' 완성도와 향상된 사용자 경험을 끌어내기 위한 미결 과제입니다.

### 2-1. 게임성 및 재미 강화 (Juice & Polish)
1. **특수 아이템(노드) 도입**: 사다리를 타다 예상치 못한 노드를 만났을 때(Swap, Teleport, Blackout 등) 규칙이 바뀌는 재미 요소 구현.
2. **시각 및 청각 피드백 연출**: 꺾이는 지점에서의 파티클 버스트(Spark), 목적지 당첨/꽝 조건에 따른 불꽃놀이 연출 및 사운드(오디오-비주얼 씽크) 연동.
3. **가시성 높은 시네마틱 줌인 카메라**: 플레이어(캐릭터)가 사다리 선을 타고 이동할 때만 부드럽게 줌-인/아웃으로 캐릭터의 시야를 추적하는 시네마틱 스크롤 구현.

### 2-2. 다회차/다인원 플레이 편의성 
1. **일괄 리스트 입력 (Batch Text Input)**: 사용자 이름을 하나씩 입력하는 불편을 없애고 직장이나 단톡방 인원 명단을 통째로 복사해 설정하는 편의 기능.
2. **사다리 맵 고유 스냅샷 공유(Seed Number)**: 특정 사다리 모양 자체를 Seed 번호로 추출해 기억해두었다가 그대로 재현할 수 있는 로직 추가.
3. **확장 가능한 무한 캔버스 이동(Pan & Zoom)**: 플레이어가 10명, 20명을 넘어서 X축 공간이 모자랄 경우 드래그로 스크롤할 수 있는 반응형 뷰포트 시스템 도입.

### 2-3. Rootfolio 연계 (Cross-Communication)
1. **웹 브라우저-플러터 메시지 브릿지**: Flutter 측의 게임 종료 결과(이벤트 등)를 `postMessage` 방식으로 부모 Webapp에 송출하여 다른 스터디 포스트가 자동으로 열리는 기능 구현.

## 🏁 3. 결론

사다리(Ghost Leg) 게임 앱은 당초 목표였던 "앱 간의 연동 실험" 및 "메인 Rootfolio의 보조 포트폴리오 컴포넌트 탑재" 목적을 달성함으로써 **현시점에서 개발의 일단락(Milestone 1 Freeze)을 선언합니다.**

차후 2.0 버전 업데이트 시에는 위의 '향후 과제'를 우선하여 실제 사용자(Player)들의 몰입감과 재미를 증진시키는 방향에 집중할 예정입니다.
