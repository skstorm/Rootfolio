# Anime Title Academy (가칭) - 일반 기획 및 마일스톤 (General Plan)

본 문서는 기획 및 디자인 관점에서 'Anime Title Academy' 앱의 컨셉, 핵심 사용자 경험(UX), 그리고 4단계 페이즈(Phase)에 따른 개발 마일스톤을 정의합니다.

---

## 🎭 1. 코어 컨셉 (Core Concept)
**"내 평범한 사진이 유머러스한 애니메이션 짤(Meme)로 탄생한다!"**
사용자가 일상에서 찍은 사진이나 갤러리 이미지를 업로드하면, 지정된 스타일(프리셋 또는 커스텀)에 맞춰 AI가 **"제목학원(유머 자막)"**을 달아주는 엔터테인먼트 앱. 최종 결과물은 복권처럼 **'스크래치'**하여 확인하는 손맛을 제공합니다.

---

## 🎯 2. 기획 전문가(Planner) 관점: 비즈니스 및 UX 플로우

### 수익 모델 (Monetization & Prompts)
앱의 생명력과 수익은 **"프롬프트 선택"** 시스템에서 나옵니다.
- **무료 프리셋 (3종)**:
    1. `다큐멘터리`: (예) "생존을 위한 극한의 사투가 시작되었다."
    2. `인터넷 밈/드립`: (예) "아 ㅋㅋ 퇴근 마려운 폼 미쳤다."
    3. `로맨스/순정`: (예) "너란 녀석... 내가 이렇게까지 해야 직성이 풀리는 건가."
- **유료/보상형 커스텀 (1+N종)**:
    - 기본 커스텀 슬롯 1개 제공. 사용자가 직접 상황이나 컨셉 입력 가능.
    - 슬롯 확장(와쿠 늘리기) 시 **인앱 결제** 또는 **동영상 리워드 광고 3회 연속 시청** 등을 요구하여 확실한 수익 창출.
- **결과물 워터마크**: 무료 이용 시 우측 하단 워터마크 강제 적용. 워터마크 제거(Pro 기능)로 과금 유도.

---

## 🎨 3. 디자인 전문가(Designer) 관점: UI 및 상호작용(Juice)

1. **테마 선택 UI**: 이미지 업로드 후 하단에 큼직하고 직관적인 가로 스크롤 카드 배치를 통해 프리셋 3종과 '커스텀(왕관 아이콘)'을 구분합니다.
2. **시각적 타이포그래피 (Styler)**: 
    - 애니메이션/예능 폰트의 핵심인 **'밝은 옐로우 + 굵고 명확한 블랙 아웃라인(Stroke)'** 텍스트 렌더링을 적용하여 B급 유머 감성을 극대화합니다.
3. **스크래치 공개 연출 (Scratch UX)**:
    - 단순히 로딩 후 이미지가 뜨는 것이 아닙니다. 
    - **Back(결과물)**: 필터+자막이 씌워진 이미지.
    - **Front(가림막)**: 변환 중이라는 UI 또는 원본 사진 그레이스케일 버전.
    - 사용자가 손가락으로 문지르면 가림막이 벗겨지고, 일정 비율(약 40%) 이상 긁어내면 **"펑!" 하는 파티클 폭발 효과**와 함께 결과물이 완전 개방됩니다.

---

## 🚀 4. 개발 마일스톤 (4-Phase Strategy)

안정적인 런칭과 모듈 결합을 위해 4단계로 나누어 점진적으로 기능을 추가합니다.

- **Phase 1: 뼈대와 코어 엔진 (MVP)**
    - 사진 입력(Picker) 및 프리셋 UI 제작.
    - Vision API 및 LLM(Gemini) 연동하여 "원본 이미지 + 애니 폰트 자막" 합성 로직 구현.
- **Phase 2: 손맛의 극대화 (Scratch UX)**
    - Phase 1 결과물을 스크래치 레이어로 덮는 뷰(View) 단위 래핑.
    - 문지르기 상호작용 및 퍼센티지(%) 연산, 폭발 애니메이션 추가.
- **Phase 3: 이미지 변환 (AI Image Gen)**
    - Stable Diffusion 등 이미지 생성 AI 파이프라인 연계.
    - 자막 생성 "이전" 단계에 원본을 애니메이션/판타지 화풍으로 변환시키는 모듈 추가.
- **Phase 4: 본격 수익화 (Ads module)**
    - 로딩 시간(Phase 3 동작 중 긴 딜레이)에 **전면 광고** 노출.
    - 커스텀 슬롯 추가, 결과 저장 시 **리워드 광고/인앱 결제** 모듈 결합.
