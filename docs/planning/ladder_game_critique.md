# 사다리타기 게임 전문가 심층 리뷰 (Expert Critique)

전문가 워크플로우(`plan-game`, `design-game`, `implement-game`)를 통해 현재 설계를 검토했을 때, **"프리미엄한 완성도"**와 **"지속 가능한 재미"**를 위해 보완하면 좋을 5가지 포인트를 정리해 보았습니다.

---

### 1. 📋 기획 전문가 (Planner): "예측 불가능한 재미"
현재 설계는 정통 사다리 규칙에 충실하지만, '게임'으로서의 의외성이 부족할 수 있습니다.
- **아이템 노드(Special Nodes) 도입**: 가로선 중간에 특수 아이템을 배치합니다.
    - `Swap`: 두 선의 위치를 강제로 바꿈.
    - `Blackout`: 일정 구간의 선을 보이지 않게 함 (안개 효과).
    - `Teleport`: 다른 수직선의 특정 지점으로 즉시 이동.
- **시드(Seed) 시스템**: 특정 사다리 모양을 '시드 번호'로 저장하여 친구와 공유하거나 동일한 맵에서 다시 대결할 수 있는 기능을 제안합니다.

### 2. 🎨 디자인 전문가 (Designer): "관조가 아닌 몰입"
현재 설계는 전체를 한눈에 보는 것에 집중되어 있습니다. 플레이어가 자신의 운명을 따라가는 '몰입감'이 필요합니다.
- **시네마틱 카메라 (Camera Follow)**: 플레이어가 이동할 때 카메라가 해당 캐릭터를 클로즈업하여 따라가는 연출을 추가합니다. (긴장감 극대화)
- **오디오 시각화 (Audio-Visual Sync)**: 선을 타고 내려갈 때 발생하는 효과음과 그에 맞춰 선이 미세하게 떨리는(Shake) 효과를 추가하여 '손맛'을 구현합니다.
- **결과 연출 (Grand Finale)**: 단순히 팝업이 뜨는 게 아니라, 목적지 도달 시 불꽃놀이 효과나 캐릭터의 승리 모션 등 확실한 보상 연출이 필요합니다.

### 3. 💻 구현 전문가 (Engineer): "대규모 데이터와 확장성"
플레이어 수가 20~30명이 넘어갈 경우를 대비해야 합니다.
- **무한 캔버스/줌 (Scalable View)**: 플레이어가 많아지면 가로로 길어지므로, 마우스 휠로 확대/축소하거나 드래그로 이동할 수 있는 뷰포트 관리가 필요합니다.
- **일괄 입력 모드 (Batch Input)**: 이름을 하나씩 입력하는 게 아니라, 텍스트를 복사/붙여넣기 하면 자동으로 플레이어 리스트를 생성하는 기능을 제안합니다.
- **공통 프로토콜 (Bridge Proto)**: 사다리 게임의 결과를 `Rootfolio` 메인 앱에 전달하여, 특정 로그나 스터디 페이지를 보상으로 연는 등 메인 웹과의 긴밀한 연동을 설계해두면 좋겠습니다.

### 4. ⚖️ 아쉬운 점 & 리스크
- **초기 성능**: Flutter Web은 초기 로딩 시 엔진을 받아오는 시간이 발생할 수 있습니다. 이를 사용자에게 세련된 '로딩 인드로'로 보여줄 기획이 필요합니다.
- **반응형 대응**: PC에서는 넓게 보이지만, 모바일 세로 모드에서는 사다리 간격이 좁아질 수 있습니다. 세로 모드 최적화 UI 설계가 추가되어야 합니다.
