## 🔥 트러블슈팅

### 1. StateNotifier → Notifier 마이그레이션 오류
- **원인**: Riverpod 3.x에서 `StateNotifier` / `StateNotifierProvider`가 deprecated 되었으나 기존 코드에서 그대로 사용
- **해결**: `ScratchNotifier`, `TitleNotifier` 를 `Notifier<T>` + `NotifierProvider`로 교체, `super(initialState)` 대신 `build()` override로 변경
- **교훈**: Riverpod 버전에 따라 API가 크게 바뀌므로 초기 의존성 추가 시 공식 docs의 최신 예시를 반드시 확인

### 2. Windows 빌드 시 NuGet / Firebase C++ SDK 오류
- **원인**: `firebase_core`, `firebase_analytics` 패키지가 Windows 빌드 시 C++ SDK를 컴파일하려 하며 NuGet.exe를 요구
- **해결**: `pubspec.yaml`에서 두 패키지를 임시 주석 처리. Phase 4에서 `google-services.json` 설정 후 재활성화 예정
- **교훈**: Firebase Flutter 패키지는 플랫폼별 네이티브 SDK를 요구함. 데스크톱 테스트 단계에서는 Firebase 없이 Mock으로 우선 진행하는 것이 효율적

### 3. Windows 개발자 모드 비활성화로 인한 빌드 실패
- **원인**: Flutter 플러그인이 심링크(symlink)를 사용하는데, Windows 개발자 모드가 꺼져 있으면 생성 불가
- **해결**: `ms-settings:developers` 를 실행해 개발자 모드 활성화
- **교훈**: Windows에서 Flutter 플러그인 포함 프로젝트 빌드 시 개발자 모드가 필수 전제 조건

### 4. `setState() called after dispose()` 경고
- **원인**: 스크래치 도중 Hot Restart가 발생하면 `_ScratchCanvasState`가 dispose 된 후에도 터치 이벤트 핸들러가 `setState`를 호출
- **해결**: `_addPoint()` 및 `onPointerUp` 에서 `setState` 호출 전 `if (mounted)` 체크 추가
- **교훈**: 비동기 또는 이벤트 기반 콜백에서는 항상 `mounted` 체크 필수

### 5. import 경로 오류 (상대 경로 depth 불일치)
- **원인**: features 하위 폴더 깊이가 서로 다른데 동일한 상대경로(`../`)로 작성
- **해결**: `flutter analyze` 출력으로 오류 파일 특정 후 정확한 depth의 상대경로로 수정
- **교훈**: 폴더 구조가 깊은 프로젝트에서는 상대경로보다 패키지 절대경로(`package:anime_title_academy/...`) 사용을 고려
