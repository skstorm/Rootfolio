# 작업일지: 2026-03-20

## 📌 완료 작업
### 1. AI 성능 로그 분석 및 병목 구간 확인
- 디버그 전용 로그를 통해 `vision_prepare`, `vision_api_call`, `vision_analysis`, `llm_generation`, `title_pipeline_total` 구간을 실측.
- `자막만 다시 생성` 시 `vision_*` 로그가 출력되지 않음을 확인하여 이미지 재해석 없이 LLM만 다시 호출되는 로직을 검증.
- 실제 병목이 Vision보다 `llm_generation` 쪽에 더 크게 분포하며, 응답 시간이 7초대에서 20초 후반까지 크게 흔들린다는 점을 확인.

### 2. 결과 화면 버튼 동작 재설계
- `다시하기` 버튼이 뒤로 가기 대신, 같은 이미지에 대해 Vision 캐시를 무시하고 새 해석 후 다시 제목을 생성하도록 변경.
- `자막만 다시 생성` 버튼을 추가하여, 이전에 분석된 태그를 재사용하고 이미지 해석 없이 자막만 다시 생성하도록 분리.
- `저장 / 공유` 버튼 위치를 우측으로 이동하고, 디버그용 `리셋 (Debug)` 버튼은 해당 버튼 상단으로 재배치.

### 3. 제목 다양성 강화 로직 추가
- 일반 생성에도 전체 태그 중 랜덤한 `핵심 태그 1~2개`를 뽑아 프롬프트에 강조 반영하도록 개선.
- `다시하기` 및 `자막만 다시 생성`에서는 최근 생성 제목 목록을 함께 보존하고, 동일한 핵심 단어/문장 구조/감정 톤을 피하도록 다양성 규칙을 추가.
- 이를 위해 `TitleViewState`에 `tags`, `styleId`, `recentTitles`를 함께 유지하도록 상태 구조를 확장.

### 4. 자막 재생성 전용 UX 분리
- `자막만 다시 생성` 동작 시 기존의 “사진을 분석하여 자막을 생성” 로딩 문구 대신, `AI가 자막을 재생성 하고 있습니다...` 문구를 별도로 표시하도록 분리.
- `TitleLoadingMode`를 도입하여 전체 파이프라인 로딩과 자막만 재생성 로딩을 구분.

### 5. Fast Prompt 실험 후 롤백
- `자막만 다시 생성` 전용 경량 Fast Prompt를 적용해 LLM 응답 시간을 단축하려는 실험을 수행.
- 실측 결과 체감 개선이 미미하여 Fast Prompt 분기를 제거하고, 일반 프롬프트 기반 재생성 경로로 롤백.
- 대신 최근 제목 회피 규칙과 핵심 태그 랜덤화는 유지하여 다양성 확보만 남기도록 정리.

## 🛠 트러블슈팅
### 1. Provider 타입 변경 후 핫리로드 런타임 충돌
- **문제**: `titleNotifierProvider`의 상태 타입을 `TitleResult?`에서 `TitleViewState?`로 변경한 뒤 핫리로드만 수행하여 런타임 타입 충돌(red screen) 발생.
- **해결**: 원인이 코드 오류가 아니라 Provider 타입 변경에 따른 런타임 캐시 충돌임을 확인하고, 핫리로드가 아닌 핫리스타트/앱 재실행이 필요함을 정리.

### 2. `자막만 다시 생성` 체감 속도 개선 실패 원인 분석
- **문제**: 이미지 해석을 생략해도 `자막만 다시 생성`이 `다시하기`와 비슷하게 느려지는 현상.
- **해결**: 로그 분석 결과 Vision 단계는 실제로 생략되고 있었으며, 지연의 주범이 LLM 응답 변동성이라는 점을 확인. 로직 오류가 아니라 LLM 병목 문제로 분류.

### 3. Fast Prompt 도입 후 효과 미미
- **문제**: 자막만 다시 생성용 경량 프롬프트를 추가했지만 체감 속도 개선이 뚜렷하지 않았고, 관리 복잡성만 증가.
- **해결**: Fast Prompt 분기를 제거하고 원래 프롬프트 체계로 복귀. 기능 실험은 롤백하되 핵심 태그 랜덤화와 최근 제목 회피 규칙은 유지.

### 4. 로딩 UI 분리 중 렌더링 분기 꼬임
- **문제**: `ResultPage`의 로딩 상태를 분리하는 과정에서 `Center` 위젯 분기가 중복되어 렌더링 로직이 꼬일 수 있는 상태 발생.
- **해결**: `TitleLoadingMode`를 기준으로 단일 로딩 분기만 남기고, 로딩 메시지만 작업 종류에 따라 바뀌도록 구조를 재정리.
