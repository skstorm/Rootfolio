- Vision 입력 이미지를 장변 `768px`, JPEG 품질 `65`로 축소하여 업로드 페이로드를 경량화.
- 자주 조정될 수 있는 AI 전처리 수치를 `AiConstants`로 분리하여 튜닝 포인트를 중앙화.
- 전처리 결과를 임시 파일로 저장하지 않고 `Uint8List`로 직접 전달하도록 변경하여 불필요한 디스크 I/O 제거.

### 4. 이미지 분석 캐시 도입

- 프롬프트 샌드박스 및 동일 이미지 반복 생성 시 체감 속도 개선 기반 확보.

### 5. 테스트 및 브랜치 정리
- `PromptTemplateService`, `TitleRepositoryImpl`, `GeminiVisionDatasource`에 대한 테스트를 추가하고 `flutter test` 통과 확인.

## 🛠 트러블슈팅
### 1. `injectable` 생성 파일 누락 및 DI 초기화 오류
- **문제**: `manual_setup.dart` 제거 후 `injection_container.config.dart`가 비어 있거나 일부 의존성이 누락되어 `GetIt.init()`가 동작하지 않음.
- **해결**: `ApiClient` 생성자 시그니처를 단순화하고, 누락된 서비스 애노테이션 및 의존성을 보강한 뒤 `build_runner`를 재실행하여 generated DI 구성을 정상화.

### 2. `dart test` 실행 환경 오류 (`dart:ui` 미지원)
- **문제**: Flutter 의존 테스트를 `dart test`로 실행하면서 `dart:ui` 관련 대량 오류 발생.
- **해결**: 해당 프로젝트는 Flutter 런타임이 필요한 테스트 구조임을 확인하고 검증 명령을 `flutter test`로 전환하여 테스트 정상 통과 확인.

### 3. Vision 성능 개선 중 임시 파일 I/O 병목 확인
- **문제**: 전처리 후 임시 JPG 파일을 저장하고 다시 읽는 구조 때문에 매 분석 요청마다 불필요한 디스크 접근이 발생.
- **해결**: `ImagePayloadPreparer`가 바이트 배열을 직접 반환하도록 변경하고, `AiClient.analyzeImage` 시그니처도 `Uint8List` 기반으로 맞춰 temp file 단계를 제거.

### 4. 캐시 도입 후 DI 생성자 불일치 오류
- **문제**: `GeminiVisionDatasource`에 캐시와 로거 의존성을 추가한 뒤 generated config와 실제 생성자 시그니처가 어긋나 정적 분석 오류 발생.
- **해결**: `build_runner` 재생성 후 `dart analyze`를 재실행해 생성 파일이 최신 생성자 인자를 반영했는지 검증하고 정상 상태로 복구.
